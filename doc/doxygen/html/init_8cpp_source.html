52 //
53 // Thread management and startup/shutdown:
54 //
55 // The network-processing threads are all part of a thread group
56 // created by AppInit() or the Qt main() function.
57 //
58 // A clean exit happens when StartShutdown() or the SIGTERM
59 // signal handler sets fRequestShutdown, which triggers
60 // the DetectShutdownThread(), which interrupts the main thread group.
61 // DetectShutdownThread() then exits, which causes AppInit() to
62 // continue (it .joins the shutdown thread).
63 // Shutdown() is then
64 // called to clean up database connections, and stop other
65 // threads that should only be stopped after the main network-processing
66 // threads have exited.
67 //
68 // Note that if running -daemon the parent process returns from AppInit2
69 // before adding any threads to the threadGroup, so .join_all() returns
70 // immediately and the parent exits from main().
71 //
72 // Shutdown for Qt is very similar, only it uses a QTimer to detect
73 // fRequestShutdown getting set, and then does the normal Qt
74 // shutdown thing.
75 //
